# 명세서: `scripts/ai_reviewer.py`

## 역할
- `ARCHITECTURE.md`의 **AI Agentic Reviewer** 원칙을 실현하기 위한 도구이다.
- 이번 변경 사항(`git diff`)중에서 대응하는 `.spec` 파일이 있는 소스 코드를 찾아, AI(Gemini 등)에게 명세 준수 여부를 검토 요청한다.

## 주요 기능 (요약)
- **변경 파일 감지**: `git diff`를 통해 `HEAD`와 이전 커밋(또는 `main` 브랜치) 사이의 변경된 `.py` 파일 목록을 추출한다.
- **명세서 매칭**: 변경된 각 소스 파일에 대해 `.spec/` 디렉터리 내의 1:1 매칭되는 `.md` 명세서를 찾아 읽어온다.
- **AI 리뷰 요청**:
  - 프롬프트: "명세서(Level 2)의 규칙들이 현재 코드(Level 3)에 정확히 반영되었는가?"
  - 소스 코드와 명세를 AI에게 전달하고, 준수/위반 여부 및 구체적인 피드백을 받는다.
- **결과 보고**: 검토 결과를 터미널에 출력하고, 중대한 위반이 발견되면 종료 코드 1을 반환하여 CI를 실패 처리한다.

## 핵심 규칙
- **환경 변수**: `GEMINI_API_KEY` (또는 기타 AI 서비스 키)를 필수로 사용한다. 키가 없으면 경고 후 종료한다.
- **명세 우선**: 코드 자체의 품질보다 **명세(Spec)와의 일치성**에 집중하여 리뷰한다.
- **포용성**: 명세서가 없는 파일은 일반적인 린트 규칙 검토로 대체하거나 건너뛴다.
- **결과 형식**: GitHub Actions의 `Step Summary`나 PR 코멘트에 기록하기 쉬운 Markdown 형식의 출력을 지향한다.
