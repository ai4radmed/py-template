# 명세서: `scripts/setup/setup_env.py`

## 역할
- 프로젝트 루트에서 `.env.example` 을 `.env` 로 복사하고, 프로젝트명/루트 경로/OS 별 기본 `LOG_PATH` 값을 자동 치환하는 편의 스크립트이다.

## 동작 (요약)
- 현재 작업 디렉터리를 프로젝트 루트로 간주하고, `.env.example` 이 없으면 에러 메시지 출력 후 종료한다.
- `.env.example` 을 `.env` 로 복사한 뒤:
  - `PROJECT_NAME` 을 현재 폴더명으로,
  - `PROJECT_ROOT` 를 현재 루트 절대 경로로,
  - `LOG_PATH` 를 OS 별 기본 추천 경로로(예: Linux `/var/log/{PROJECT_NAME}`) 치환해 저장한다.
- 마지막으로 `_environment` 파일을 `.env` 에서 복사 생성 시도(실패해도 치명적이지 않음) 한다.

## 핵심 규칙
- 업계 표준인 “사용자가 `.env.example` 을 수동 복사·편집” 패턴 위에 올라가는 **선택적** 편의 도구이며, 사용하지 않아도 된다.
- 실행 위치는 프로젝트 루트여야 하며, 스크립트는 로컬 개발 환경 초기화 용도로만 사용한다.
