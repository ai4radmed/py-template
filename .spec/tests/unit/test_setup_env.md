# 명세서: `tests/unit/test_setup_env.py`

## 역할
- `scripts/setup/setup_env.py` 스크립트가 `.env.example` 을 기반으로 `.env` 를 올바르게 생성·치환하는지 검증한다.

## 테스트 범위 (요약)
- `.env.example` 이 존재하는 경우:
  - `PROJECT_NAME` 이 현재 디렉터리명으로 치환되는지 확인.
  - `PROJECT_ROOT` 가 현재 루트 절대 경로로 설정되는지 확인.
  - `LOG_PATH` 가 OS 별 기본 경로 포맷(예: `/var/log/{PROJECT_NAME}` 등)을 따르며 `{PROJECT_NAME}` 플레이스홀더를 포함하는지 확인.
  - 기타 키(예: `OTHER_KEY`) 는 보존되는지 확인.
- `.env.example` 이 없는 경우:
  - `main()` 이 실패 코드(1)를 반환하고, `.env` 파일을 생성하지 않는지 확인.

## 핵심 규칙
- 테스트는 실제 파일 시스템의 임시 디렉터리(`tmp_path`) 를 사용하여, 프로젝트 루트 기준의 상대 경로 동작을 그대로 재현한다.
- 현재 작업 디렉터리(`cwd`) 를 변경해 실행한 뒤 반드시 원래 위치로 복원해야 하며, 전역 상태 오염을 남기지 않는다.
- **불필요한 임포트 제거**: `pytest` 모듈이 직접 호출되지 않는 파일에서는 임포트를 제거한다.
