# 명세서: `tests/e2e/` 디렉터리

## 역할
- 실제 사용자 시나리오 단위의 **종단 간(E2E) 테스트**를 모아두는 디렉터리이다.
- 애플리케이션이 여러 모듈과 외부 시스템을 아우르는 전체 플로우에서 기대한 대로 동작하는지 검증한다.

## 구조 및 규칙
- 파일 위치:
  - 모든 E2E 테스트는 `tests/e2e/` 또는 하위 서브디렉터리에 위치시킨다.
- 네이밍:
  - `test_*.py` 형태의 pytest 모듈(또는 별도의 E2E 러너)을 사용한다.
  - 주요 시나리오는 `specs/features/` 와 1:1로 매핑되는 것을 목표로 한다(예: 로그인, 보고서 생성 등).
- 책임:
  - 환경 의존성이 큰 만큼, E2E 테스트는 기본적으로 선택적(optional) 실행 대상으로 취급하며,
    CI 에서는 태그/마커를 통해 명시적으로 포함/제외한다.
  - 테스트 전에 필요한 초기 상태(데이터 시드, 설정 파일, 서비스 기동 등)를 명확히 정의하고,
    종료 후에는 가능한 한 원복(cleanup) 한다.
