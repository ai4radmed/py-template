# 명세서: `tests/integration/` 디렉터리

## 역할
- 모듈 간 연동(데이터베이스, 파일 IO, 외부 시스템 포함)을 검증하는 **통합 테스트**들을 모아두는 디렉터리이다.
- 단위 테스트(`tests/unit/`) 에서 검증하기 어려운 실제 의존성 간 상호작용을 검증한다.

## 구조 및 규칙
- 파일 위치:
  - 모든 통합 테스트는 `tests/integration/` 또는 하위 서브디렉터리에 위치시킨다.
- 네이밍:
  - `test_*.py` 형태의 pytest 모듈을 사용한다.
  - 각 테스트 파일에 대해 필요하다면 `.spec/tests/integration/test_*.md` 형태의 개별 명세서를 추가해도 좋다.
- 책임:
  - 실제 DB, 파일 시스템, 외부 API 등을 사용하는 경우, 테스트 전/후 리소스 정리(fixtures, tmp_path 등)를 명확히 정의한다.
  - 외부 시스템이 없는 환경에서도 실행 가능하도록, 가능하면 모킹/로컬 더미 서비스를 사용하는 것을 우선 고려한다.
